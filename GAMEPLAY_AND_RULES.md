# 卡牌設定規則與遊戲玩法

本文件說明本專案中**卡牌資料如何定義**、**組牌規則**，以及**對戰時玩家會遇到的流程與機制**。數值與演出若以場景／Prefab 上設定為準，與程式預設不同時，以遊戲內表現為準。

---

## 一、卡牌資料從哪裡來

- 主要表檔路徑：`Assets/Assets/Datas/CardList.csv`
- **怪獸列**：`monster,<編號>,<卡名>,<英文名>,<攻擊力>,<生命值>`
- **法術列**：`spell,<三位編號>,<卡名>,<英文名>,<效果說明>`（效果欄為給玩家與企劃閱讀的敘述，實際結算以程式邏輯為準）

新增或調整卡牌時，應同步確認遊戲內載入流程是否會讀取到該列（避免僅改 CSV 卻未匯入／快取未更新）。

---

## 二、卡牌類型與編號（程式內）

- **怪獸牌**：使用 **非負整數**編號，對應 CSV 的怪獸編號。
- **法術牌**：在存檔與牌組結構中，法術以 **負數鍵** 表示，避免與怪獸編號衝突（見 `DeckCardId`：依法術序號換算）。  
  玩家在遊戲中通常只看「卡名與效果」，不需記住內部鍵。

---

## 三、組牌（ Buildbeck／牌組編輯）規則

| 項目 | 說明 |
|------|------|
| **牌組張數上限** | 預設 **30 張**（`DeckManager.maxDeckCards`，可在 Inspector 調整）。 |
| **槽位** | 可有多組牌組槽；可為槽位設定**顯示名稱**並**儲存**；也可**解散／清空**該槽。 |
| **持有與編組** | 僅能使用你已「持有」的卡來組牌（實際檢查方式依商店／背包與存檔狀態）；編輯介面會區分持有張數與已放入牌組張數。 |

同一張卡的重複張數可否超過持有量，以遊戲內按鈕與提示為準。

### 3.1 組牌介面美術：縮圖與立繪

| 用途 | 資源位置（專案內） | 說明 |
|------|---------------------|------|
| **組牌／Buildbeck 左側館藏格** | `Assets/UI/DeckThumb/` | 館藏列表格子顯示**縮圖**（與 CSV／`CardStore` 中為該卡設定的縮圖欄對應）。 |
| **本體立繪（商店／背包預覽等）** | `Assets/UI/CardArt/` | 較大張的**卡牌立繪**；不得與縮圖混用語意時，企劃與美術宜分資料夾管理。 |

匯入圖檔時，可依 Editor 既有 **自動綁定**流程寫回場景／Prefab 上的 `CardStore` 覆寫資料（細節以 `Assets/Editor/CardArtworkAutoBinder.cs` 與資產工作流程為準）。改版新增卡時請**同時**補齊縮圖與立繪（若有其中一張缺漏，館藏格或詳情端可能退回僅顯示另一類圖或留白，以實機為準）。

## 四、對戰玩法概覽

### 4.1 勝負條件

- **雙方各有「英雄」生命值**；生命值先降至 **0** 的一方落敗。
- 預設起始英雄生命為 **20**（`BattleSimulationManager.startHealth`，可在場景中調整）。

### 4.2 開場與手牌

- 開場會以**骰子／開場演出**決定**先手**，並有一段開場展示時間。
- 起手抽牌數為 **5**，且不會超過**手牌上限**（預設手牌上限 **7** 張：`maxHandSize`）。

### 4.3 回合與行動（概念）

- 回合中通常可：**打出怪獸或符合條件的法術**、進行**攻擊**、必要時**結束回合**。
- **場上怪獸區**：同一側同一時間主要為**單一怪獸**與玩法規則互動（例如部分法術僅在「場上已有怪獸」時可從手牌打出）。
- **法術**：例如 **火球術**（對場上或英雄的傷害規則）、**初級治療**（治療我方場上怪獸；場上已有怪時，部分其他手牌可能無法打出）、**林可的凝視**（置場型特殊規則與禁止攻擊等）——細節以對戰內提示與結算為準。

### 4.4 天氣系統

對戰進行一段時間後會進入**天氣預報／天氣生效**節奏（**第一回合不會觸發天氣預報**）。常見四種天氣名稱與效果概念如下（完整文案見場上 UI）：

| 天氣 | 效果概念（摘要） |
|------|-------------------|
| **緋焰時雨** | 回合結束時，雙方**場上怪獸**受到固定傷害（英雄不受此項）。 |
| **月華聖祈** | 本回合**治療量額外加成**。 |
| **蒼潮夜湧** | 本回合**直接攻擊英雄**的傷害減免。 |
| **朔風森詠** | 本回合**第一張法術**效果強化（回合內觸發過後會消耗該加成機會）。 |

天氣實際**持續回合數**、**冷卻**與是否優先指定第一輪天氣等，由場景中的戰鬥設定控制。

### 4.5 對戰畫面的顯示比例（設定場景套用）

- 玩家在 **Settings**（遊戲設定）選擇的「顯示比例」預設，會在進入對戰等流程時套用至 **Battle Simulation** 相關的卡牌版面／文字／場上配置（細項以程式與 Prefab 為準）。
- 預設表（JSON）：`Assets/Resources/BattleCardTuningPresets.json`  
  （含「預設一」「對照組」等項目，供對照與驗證；實際遊玩以玩家在設定場景選定並儲存的選項為準。）
- UI 上以 **對照表** 呈現之手牌／場上參數，數值單位與欄位意義以遊戲內顯示與上述 JSON 註解／Tooltips 為準。

## 五、遊戲設定場景（Settings）

- **進入方式**：多數場景之全域選單內的「**遊戲設定**」（場景檔：`Assets/Scenes/Settings.unity`）。
- **顯示比例**：選擇戰鬥場景卡牌調校預設；目前選擇以 **PlayerPrefs** 鍵 `battle_card_tuning_preset_id` 保存，進入對戰時由程式讀取並套用（對照表細節見上文 4.5）。
- **畫面品質**：切換 Unity Quality 等級，以 PlayerPrefs 鍵 `settings_quality_level` 記住。

## 六、卡牌熟練度與怪物技能

本作以**卡牌熟練度**分階解鎖怪物牌技能（見 [`CARD_PROFICIENCY_GDD.md`](CARD_PROFICIENCY_GDD.md)）：toward-B 進度滿 **2.0**（不限難度）→ **B**（約 10 場勝）；**10 場普通難度勝利** → **C**。單局結算：勝 +1/2、平 +2/3×1/2、敗 +1/3×1/2。**國王／王后／民兵** 為御三家，預設 **C**。對戰結束浮層會列出牌組內怪物牌並播放熟練度條動畫。

| 階段 | 玩家體驗（摘要） |
|------|------------------|
| **A** | 無技能；可提示「熟練後解鎖」 |
| **B** | 一行技能摘要 |
| **C** | 完整技能說明 |

完整企劃、解鎖條件類型、待決事項與 UI 原則見：**[`CARD_PROFICIENCY_GDD.md`](CARD_PROFICIENCY_GDD.md)**。  
單卡戰技 A/B/C 文案（含**國王**範例）見：**[`卡牌技能階段式揭露.md`](卡牌技能階段式揭露.md)**。  
市場竞品與引用來源見：**[`MARKET_ANALYSIS_SOURCES.md`](MARKET_ANALYSIS_SOURCES.md)**。
